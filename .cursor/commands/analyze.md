# analyze

/analyze は「調査・解析のみ」を行う。実装・コード変更・diff提示・置換ブロック提示・ファイル編集・コミット/Push手順は一切禁止。

やること（この順で出力）：
1) 依頼の要約（1〜2行）
2) 関連しそうな箇所（ファイル/関数/設定/外部依存）を列挙
3) 原因仮説を1〜5個（確度: 高/中/低）
4) 各仮説の確認方法（ログ/再現手順/確認コマンド/観測ポイント）
5) 修正案の方向性（最大3案）※コードは書かない。触る範囲・リスク・副作用を書く
6) 次にユーザーが貼るべき情報（必要最小限）

ルール：
- 参照順は CLAUDE.md → docs/PROJECT_OVERVIEW.md → TODO.md → 既存コード。
- 事実と推測は分けて書く（推測には「推測」と付ける）。
- 質問は「判断不能な致命点」のみに絞る。
